# Kiko Sushi — النسخة الحقيقية (Firebase)

هذي نسخة النظام اللي تشتغل فعليًا بين أجهزة مختلفة: تحدّث حالة الطلب من جهازك، وتنعكس فورًا عند العميل على جواله — لأن البيانات صارت محفوظة في قاعدة بيانات حقيقية (Firestore) بدل المتصفح المحلي.

## الملفات

- `index.html` — النظام كامل (لوحة التحكم + صفحة طلب العميل)
- `firebase-config.js` — هنا تحط بيانات مشروعك في Firebase (خطوات تحت)
- `firestore.rules` — قواعد أمان قاعدة البيانات
- `firebase.json` / `.firebaserc` — إعدادات الاستضافة على Firebase Hosting

## الخطوة 1: إنشاء مشروع Firebase (مجاني)

1. افتح https://console.firebase.google.com
2. اضغط **Add project** واختر اسم (مثلاً kiko-sushi)
3. من القائمة الجانبية: **Build → Firestore Database → Create database**
   - اختر أي منطقة قريبة (مثلاً `europe-west` أو `me-central1` إذا متاحة)
   - اختر **Start in production mode**
4. من الصفحة الرئيسية للمشروع، اضغط أيقونة **`</>`** (Web) لإضافة تطبيق ويب جديد
   - أعطه أي اسم واضغط **Register app**
   - بيطلع لك كائن `firebaseConfig` فيه قيم مثل `apiKey`, `projectId`... انسخه

## الخطوة 2: تعبئة `firebase-config.js`

افتح ملف `firebase-config.js` والصق القيم اللي نسختها بدل `"ضع_قيمتك_هنا"` في كل سطر.

## الخطوة 3: رفع قواعد الأمان

قواعد الأمان في `firestore.rules` تحصر الوصول في وثيقة النظام فقط (`kiko_sushi/store`) ولا تسمح بأي شي آخر. من داخل Firebase Console:
**Firestore Database → Rules** → الصق محتوى `firestore.rules` واضغط **Publish**.

⚠️ **ملاحظة أمان مهمة:** بما إن النظام حاليًا بدون تسجيل دخول (لا لك ولا للعميل)، قواعد الأمان مضطرة تسمح بالقراءة والكتابة لأي شخص يعرف رابط الوثيقة تحديدًا. هذا يعني من الناحية النظرية أي شخص عنده معرفة تقنية يقدر يوصل لبيانات نظامك ويعدلها مباشرة عبر Firestore API (مو بس من الواجهة). إذا حبيت حماية أقوى مستقبلًا (تسجيل دخول لك كصاحب مشروع، وربط كل طلب برقم جوال العميل فقط)، قولي وأضيفها.

## الخطوة 4: تجربة محلية (قبل الرفع)

ES Modules ما تشتغل من ملف مفتوح مباشرة (`file://`) — تحتاج سيرفر بسيط:

```bash
cd kiko-sushi-firebase
python3 -m http.server 8000
```

ثم افتح `http://localhost:8000` بالمتصفح.

## الخطوة 5: النشر الفعلي على الإنترنت (Firebase Hosting)

على جهازك (يحتاج Node.js مثبت):

```bash
npm install -g firebase-tools
firebase login
cd kiko-sushi-firebase
firebase use --add        # اختر مشروعك اللي أنشأته بالخطوة 1
firebase deploy
```

بعد الانتهاء بيعطيك رابط حقيقي زي:
`https://kiko-sushi.web.app`

هذا الرابط + `#order` (مثال: `https://kiko-sushi.web.app/#order`) هو اللي ترسله للعملاء عبر واتساب — وبيشتغل فعليًا من أي جوال.

## ملاحظات

- كل التعديلات (مخزون، أصناف، طلبات، مصاريف) تُحفظ الآن في Firestore مباشرة وتظهر فورًا لأي شخص فاتح النظام بنفس اللحظة (لوحتك + صفحة العميل).
- لو حبيت تضيف حسابات دخول (لك أنت، أو لموظفين إضافيين)، أو تحويل الأمان لمستوى أعلى، قولي وأبنيها كخطوة تالية.
